# wiki-query

위키를 탐색하여 질문에 답하고, 가치 있는 결과를 위키에 저장합니다.

## 사용법

```
/wiki-query [질문]
```

예) `/wiki-query 머신러닝과 딥러닝의 차이점은?`

## 실행 절차

인자로 받은 질문(또는 대화에서 이어지는 질문)을 대상으로 아래 순서대로 진행합니다.

1. **위키 현황 파악**
   - `wiki/index.md`를 읽어 관련 페이지 후보를 파악합니다.

2. **관련 페이지 읽기**
   - index.md에서 찾은 관련 페이지들을 읽습니다.
   - 필요시 추가 페이지를 재귀적으로 탐색합니다.

3. **답변 합성**
   - 읽은 위키 페이지를 기반으로 답변을 작성합니다.
   - 각 클레임에 출처 페이지를 `([[페이지명]])` 형식으로 명시합니다.
   - 위키에 해당 정보가 없으면 솔직하게 "위키에 아직 해당 내용이 없습니다"라고 답합니다.

4. **가치 있는 탐색 저장** (해당하는 경우)
   - 비교 분석, 새로운 연결, 심층 탐색 결과는 `wiki/analyses/[주제].md`로 저장합니다.
   - 사용자에게 저장 여부를 확인한 후 진행합니다.

5. **log.md 업데이트** (최상단에 prepend)
   ```
   ## [오늘날짜] query | 질문 요약
   - 참조 페이지: [목록]
   - 답변 요약 또는 생성된 페이지: [내용]
   ```
