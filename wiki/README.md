# LLM Wiki

Andrej Karpathy의 [LLM Wiki 패턴](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)을 기반으로 한 개인 지식 베이스입니다.

## 개념

RAG처럼 매번 원본 문서에서 답을 재도출하는 대신, LLM이 **지속적으로 위키를 구축하고 유지**합니다. 새 소스를 추가할 때마다 LLM이 관련 위키 페이지를 업데이트하고 교차 참조를 유지합니다. 지식이 쌓일수록 위키는 점점 풍부해집니다.

## 구조

```
./
├── CLAUDE.md          # LLM 행동 지침 (스키마)
├── README.md          # 이 파일
├── raw/               # 원본 소스 (불변)
│   └── assets/        # 이미지 및 첨부파일
└── wiki/              # LLM이 유지하는 지식 베이스
    ├── index.md       # 전체 카탈로그
    ├── log.md         # 작업 이력
    ├── sources/       # 소스별 요약
    ├── entities/      # 인물, 조직, 제품
    ├── concepts/      # 개념, 이론, 기술
    └── analyses/      # 비교, 분석, 합성
```

## 스킬 (슬래시 커맨드)

이 프로젝트는 세 가지 Claude Code 슬래시 커맨드를 제공합니다.

### `/wiki-ingest [파일경로]`

`raw/`에 추가된 소스를 위키에 통합합니다.

```
/wiki-ingest raw/article.md
```

- 소스를 읽고 핵심 정보를 추출합니다.
- `wiki/sources/`에 요약 페이지를 생성합니다.
- 관련 엔티티·개념 페이지를 업데이트합니다 (최대 10-15개).
- `wiki/index.md`와 `wiki/log.md`를 자동 갱신합니다.

### `/wiki-query [질문]`

위키를 탐색하여 질문에 답하고 가치 있는 탐색 결과를 저장합니다.

```
/wiki-query 머신러닝과 딥러닝의 차이점은?
```

- `wiki/index.md`에서 관련 페이지를 찾아 읽습니다.
- 출처를 명시한 합성 답변을 제공합니다.
- 비교 분석 등 가치 있는 결과는 `wiki/analyses/`에 저장합니다.

### `/wiki-lint`

위키 전체를 점검하여 품질을 유지합니다.

```
/wiki-lint
```

- 모순된 클레임, 고립 페이지, 깨진 링크를 탐지합니다.
- 오래된 정보 및 누락된 교차 참조를 찾아냅니다.
- 문제 목록을 보고한 후 수정을 적용합니다.

## 선택 사항: CLI 도구

위키가 커지면 LLM이 더 효율적으로 동작하도록 도와주는 소형 도구를 추가할 수 있습니다. 가장 유용한 것은 위키 페이지를 대상으로 한 **검색 엔진**입니다. 소규모에서는 `index.md`로 충분하지만, 위키가 성장하면 제대로 된 검색이 필요해집니다.

[qmd](https://github.com/tobi/qmd)가 좋은 선택입니다. 마크다운 파일을 위한 로컬 검색 엔진으로, BM25/벡터 하이브리드 검색과 LLM 재순위화를 모두 온디바이스로 제공합니다. CLI(LLM이 셸에서 호출 가능)와 MCP 서버(LLM이 네이티브 도구로 사용 가능) 두 가지 인터페이스를 모두 지원합니다.

더 단순한 도구를 직접 만들 수도 있습니다. 필요가 생기면 LLM에게 간단한 검색 스크립트를 바이브코딩해 달라고 요청하세요.

## 팁과 트릭

- **Obsidian Web Clipper**는 웹 아티클을 마크다운으로 변환하는 브라우저 확장입니다. `raw/`에 소스를 빠르게 추가할 때 매우 유용합니다.

- **이미지를 로컬에 저장하세요.** Obsidian 설정 → Files and links에서 "Attachment folder path"를 `raw/assets/`로 지정합니다. 그런 다음 설정 → Hotkeys에서 "Download attachments for current file"을 단축키(예: `Ctrl+Shift+D`)에 등록합니다. 아티클을 클리핑한 후 단축키를 누르면 모든 이미지가 로컬에 저장됩니다. LLM이 깨질 수 있는 URL 대신 이미지를 직접 참조할 수 있어 유용합니다. 단, LLM은 인라인 이미지가 포함된 마크다운을 한 번에 읽지 못하므로, 텍스트를 먼저 읽은 후 필요한 이미지를 별도로 확인하는 방식으로 작동합니다.

- **Obsidian 그래프 뷰**는 위키의 전체 구조를 파악하는 가장 좋은 방법입니다. 무엇이 무엇과 연결되어 있는지, 어떤 페이지가 허브이고 어떤 페이지가 고립되어 있는지 한눈에 볼 수 있습니다.

- **Marp**는 마크다운 기반 슬라이드 덱 형식입니다. Obsidian 플러그인으로 제공됩니다. 위키 콘텐츠에서 직접 프레젠테이션을 생성할 때 유용합니다.

- **Dataview**는 페이지 frontmatter를 쿼리하는 Obsidian 플러그인입니다. LLM이 위키 페이지에 YAML frontmatter(태그, 날짜, 소스 수 등)를 추가하면, Dataview로 동적 테이블과 목록을 생성할 수 있습니다.

- 위키는 마크다운 파일로 구성된 **git 저장소**입니다. 버전 히스토리, 브랜치, 협업 기능을 무료로 사용할 수 있습니다.
