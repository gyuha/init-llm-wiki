# LLM Wiki

Andrej Karpathy의 [LLM Wiki 패턴](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)을 기반으로 한 개인 지식 베이스입니다.

## 개념

RAG처럼 매번 원본 문서에서 답을 재도출하는 대신, LLM이 **지속적으로 위키를 구축하고 유지**합니다. 새 소스를 추가할 때마다 LLM이 관련 위키 페이지를 업데이트하고 교차 참조를 유지합니다. 지식이 쌓일수록 위키는 점점 풍부해집니다.

## 구조

```
./
├── CLAUDE.md          # LLM 행동 지침 (스키마)
├── README.md          # 이 파일
├── raw/               # 원본 소스 (불변)
│   └── assets/        # 이미지 및 첨부파일
└── wiki/              # LLM이 유지하는 지식 베이스
    ├── index.md       # 전체 카탈로그
    ├── log.md         # 작업 이력
    ├── sources/       # 소스별 요약
    ├── entities/      # 인물, 조직, 제품
    ├── concepts/      # 개념, 이론, 기술
    └── analyses/      # 비교, 분석, 합성
```

## 사용 방법

### 소스 추가 (Ingest)

1. `raw/` 디렉토리에 마크다운 파일을 추가합니다.
   - Obsidian Web Clipper로 웹 아티클을 클리핑하면 편리합니다.
2. Claude에게 요청합니다: _"raw/파일명.md를 ingest해줘"_
3. LLM이 소스를 읽고 관련 위키 페이지를 업데이트합니다.

### 질문 (Query)

- Claude에게 직접 질문합니다.
- LLM이 위키를 탐색하여 합성된 답변을 제공합니다.
- 가치 있는 답변은 위키 페이지로 저장됩니다.

### 위키 점검 (Lint)

- _"위키 건강 검사 해줘"_ 라고 요청합니다.
- LLM이 모순, 고립 페이지, 오래된 정보 등을 점검합니다.

## 철학

> "LLM은 지루해하지 않고, 교차 참조 업데이트를 잊지 않으며, 한 번에 15개의 파일을 수정할 수 있다."
> — Andrej Karpathy

인간의 역할: 소스 큐레이션, 방향 설정, 좋은 질문하기
LLM의 역할: 요약, 교차 참조, 정리, 유지보수
