# LLM Wiki

Andrej Karpathy의 [LLM Wiki 패턴](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)을 기반으로 한 개인 지식 베이스입니다.

## 개념

RAG처럼 매번 원본 문서에서 답을 재도출하는 대신, LLM이 **지속적으로 위키를 구축하고 유지**합니다. 새 소스를 추가할 때마다 LLM이 관련 위키 페이지를 업데이트하고 교차 참조를 유지합니다. 지식이 쌓일수록 위키는 점점 풍부해집니다.

## 구조

```
./
├── CLAUDE.md          # LLM 행동 지침 (스키마)
├── README.md          # 이 파일
├── raw/               # 원본 소스 (불변)
│   └── assets/        # 이미지 및 첨부파일
└── wiki/              # LLM이 유지하는 지식 베이스
    ├── index.md       # 전체 카탈로그
    ├── log.md         # 작업 이력
    ├── sources/       # 소스별 요약
    ├── entities/      # 인물, 조직, 제품
    ├── concepts/      # 개념, 이론, 기술
    └── analyses/      # 비교, 분석, 합성
```

## 스킬 (슬래시 커맨드)

이 프로젝트는 세 가지 Claude Code 슬래시 커맨드를 제공합니다.

### `/wiki-ingest [파일경로]`

`raw/`에 추가된 소스를 위키에 통합합니다.

```
/wiki-ingest raw/article.md
```

- 소스를 읽고 핵심 정보를 추출합니다.
- `wiki/sources/`에 요약 페이지를 생성합니다.
- 관련 엔티티·개념 페이지를 업데이트합니다 (최대 10-15개).
- `wiki/index.md`와 `wiki/log.md`를 자동 갱신합니다.

### `/wiki-query [질문]`

위키를 탐색하여 질문에 답하고 가치 있는 탐색 결과를 저장합니다.

```
/wiki-query 머신러닝과 딥러닝의 차이점은?
```

- `wiki/index.md`에서 관련 페이지를 찾아 읽습니다.
- 출처를 명시한 합성 답변을 제공합니다.
- 비교 분석 등 가치 있는 결과는 `wiki/analyses/`에 저장합니다.

### `/wiki-lint`

위키 전체를 점검하여 품질을 유지합니다.

```
/wiki-lint
```

- 모순된 클레임, 고립 페이지, 깨진 링크를 탐지합니다.
- 오래된 정보 및 누락된 교차 참조를 찾아냅니다.
- 문제 목록을 보고한 후 수정을 적용합니다.

## 철학

> "LLM은 지루해하지 않고, 교차 참조 업데이트를 잊지 않으며, 한 번에 15개의 파일을 수정할 수 있다."
> — Andrej Karpathy

인간의 역할: 소스 큐레이션, 방향 설정, 좋은 질문하기
LLM의 역할: 요약, 교차 참조, 정리, 유지보수
