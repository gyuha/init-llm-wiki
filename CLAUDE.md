# LLM Wiki Schema

이 파일은 LLM(Claude)이 이 위키를 어떻게 구성하고, 유지하고, 발전시켜야 하는지를 정의하는 스키마 문서입니다.
인간은 소스를 큐레이션하고 질문을 던집니다. LLM은 나머지 모든 작업(요약, 교차 참조, 정리, 색인)을 담당합니다.

---

## 디렉토리 구조

```
./
├── CLAUDE.md          # 이 파일 - LLM 행동 지침 (스키마)
├── README.md          # 프로젝트 소개
├── raw/               # 원본 소스 (불변 - LLM이 읽기만 함)
│   ├── assets/        # 다운로드된 이미지 및 첨부파일
│   └── *.md           # 클리핑한 아티클, 논문, 노트 등
└── wiki/              # LLM이 생성·유지하는 지식 베이스
    ├── index.md       # 위키 전체 카탈로그 (콘텐츠 기반)
    ├── log.md         # 작업 이력 (시간 순서 기록)
    ├── sources/       # 소스별 요약 페이지
    ├── entities/      # 인물, 조직, 제품 페이지
    ├── concepts/      # 개념, 이론, 기술 페이지
    └── analyses/      # 비교, 분석, 합성 페이지
```

---

## 핵심 원칙

1. **raw/** 디렉토리는 절대 수정하지 않습니다. 읽기만 합니다.
2. **wiki/** 디렉토리는 LLM이 완전히 소유합니다. 인간은 읽고 탐색합니다.
3. 모든 위키 페이지는 서로 교차 참조되어야 합니다. 고립된 페이지(orphan)를 만들지 않습니다.
4. 모순이 발견되면 해결하거나 명시적으로 표시합니다.
5. 모든 클레임에는 출처 소스가 명시되어야 합니다.

## 워크플로우

워크플로우는 슬래시 커맨드 스킬로 구현되어 있습니다. 자세한 절차는 각 스킬 파일을 참조하세요.

| 커맨드 | 스킬 파일 | 설명 |
|--------|-----------|------|
| `/wiki-ingest [파일경로]` | `.claude/commands/wiki-ingest.md` | 새 소스를 위키에 통합 |
| `/wiki-query [질문]` | `.claude/commands/wiki-query.md` | 위키를 탐색하여 답변 합성 |
| `/wiki-lint` | `.claude/commands/wiki-lint.md` | 위키 건강 검사 및 수정 |

## 위키 페이지 규칙

### 파일명
- 소문자, 하이픈 구분: `machine-learning.md`, `john-doe.md`
- 카테고리별 하위 디렉토리 사용:
  - `wiki/sources/` - 소스 요약
  - `wiki/entities/` - 인물, 조직, 제품
  - `wiki/concepts/` - 아이디어, 이론, 기술
  - `wiki/analyses/` - 비교, 분석, 합성 결과

### 페이지 형식

```markdown
---
title: 페이지 제목
type: entity | concept | source | analysis
tags: [태그1, 태그2]
sources: [소스파일명1.md, 소스파일명2.md]
updated: YYYY-MM-DD
---

# 페이지 제목

한 줄 요약.

## 내용

...

## 관련 페이지

- [[관련-페이지-1]] - 관계 설명
- [[관련-페이지-2]] - 관계 설명

## 출처

- `raw/소스파일.md` - 인용 내용
```

### 교차 참조
- Obsidian 위키 링크 형식 사용: `[[페이지명]]`
- 링크 시 관계를 설명하는 텍스트 추가 권장

## index.md 형식

```markdown
# Wiki Index

마지막 업데이트: YYYY-MM-DD | 총 N개 페이지

## Sources (소스 요약)
- [[sources/소스명]] — 한 줄 설명 | 날짜

## Entities (엔티티)
- [[entities/이름]] — 한 줄 설명

## Concepts (개념)
- [[concepts/개념명]] — 한 줄 설명

## Analyses (분석/합성)
- [[analyses/분석명]] — 한 줄 설명
```

## log.md 형식

항목은 최신 순서로 **상단에 prepend**합니다.

```markdown
## [YYYY-MM-DD] ingest | 제목
- 소스: raw/파일명.md
- 생성/수정 페이지: [목록]
- 주요 업데이트: [요약]

## [YYYY-MM-DD] query | 질문 요약
- 참조 페이지: [목록]
- 결과: [요약 또는 생성된 페이지명]

## [YYYY-MM-DD] lint
- 점검 페이지 수: N
- 발견된 문제: [요약]
- 수정된 항목: [목록]
```

파싱 팁: `grep "^## \[" wiki/log.md | head -5` 로 최근 5개 항목을 확인할 수 있습니다.

## 세션 시작 시 체크리스트

새 대화를 시작할 때:
1. `CLAUDE.md` (이 파일)를 읽어 현재 스키마를 파악합니다.
2. `wiki/index.md`를 읽어 위키 현황을 파악합니다.
3. `wiki/log.md`의 최근 항목을 읽어 이전 세션에서 무엇이 됐는지 확인합니다.
4. 인간의 지시를 기다립니다.
