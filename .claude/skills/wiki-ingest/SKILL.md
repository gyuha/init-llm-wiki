# wiki-ingest

새 소스 파일을 위키에 통합합니다.

## 사용법

```
/wiki-ingest [파일경로]
```

예) `/wiki-ingest raw/article.md`

## 실행 절차

인자로 받은 파일경로(또는 대화에서 언급된 파일)를 대상으로 아래 순서대로 진행합니다.

1. **CLAUDE.md 및 위키 현황 파악**
   - `CLAUDE.md`를 읽어 스키마를 확인합니다.
   - `wiki/index.md`를 읽어 기존 위키 구조를 파악합니다.
   - `wiki/log.md` 최근 항목을 읽어 이전 작업을 확인합니다.

2. **소스 읽기**
   - 지정된 `raw/` 파일을 전체 읽습니다.
   - 핵심 정보, 주요 엔티티, 개념, 주장을 파악합니다.

3. **소스 요약 페이지 생성**
   - `wiki/sources/[파일명].md`를 생성합니다.
   - CLAUDE.md의 페이지 형식(frontmatter 포함)을 따릅니다.

4. **관련 위키 페이지 업데이트**
   - 영향받는 엔티티 페이지(`wiki/entities/`)를 업데이트하거나 신규 생성합니다.
   - 영향받는 개념 페이지(`wiki/concepts/`)를 업데이트하거나 신규 생성합니다.
   - 기존 클레임과 모순이 있으면 해당 페이지에 명시합니다.
   - 교차 참조(`[[페이지명]]`)를 추가합니다.
   - 한 번에 최대 10-15개 페이지를 수정합니다.

5. **index.md 업데이트**
   - 신규 및 변경된 페이지를 반영합니다.
   - 날짜와 총 페이지 수를 갱신합니다.

6. **log.md 업데이트** (최상단에 prepend)
   ```
   ## [오늘날짜] ingest | 소스 제목
   - 소스: raw/파일명.md
   - 생성/수정 페이지: [목록]
   - 주요 업데이트: [요약]
   ```

7. **완료 보고**
   - 생성/수정된 페이지 목록을 요약하여 보고합니다.
