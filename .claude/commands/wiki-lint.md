위키 전체를 점검하여 모순, 고립 페이지, 오래된 정보, 누락된 교차 참조를 찾아 수정합니다.

## 사용법

```
/wiki-lint
```

## 실행 절차

1. **위키 전체 읽기**
   - `wiki/index.md`를 읽어 모든 페이지 목록을 확보합니다.
   - 모든 위키 페이지를 순서대로 읽습니다.

2. **문제 탐지**
   다음 항목들을 점검합니다:

   - **모순**: 두 페이지 간에 서로 다른 클레임이 있는 경우
   - **고립 페이지(orphan)**: 어떤 페이지에서도 링크되지 않은 페이지
   - **오래된 정보**: 최신 소스에 의해 supersede된 클레임
   - **미생성 개념**: 여러 페이지에서 언급되지만 자체 페이지가 없는 중요 개념
   - **누락된 교차 참조**: 명백히 연결되어야 할 페이지들 간의 링크 부재
   - **깨진 링크**: 존재하지 않는 페이지를 가리키는 `[[링크]]`
   - **frontmatter 이상**: 누락되거나 잘못된 메타데이터

3. **문제 보고**
   발견된 문제를 카테고리별로 정리하여 보고합니다:
   ```
   ## Lint 결과

   ### 모순
   - [페이지A] vs [페이지B]: 내용

   ### 고립 페이지
   - [페이지명]: 권장 연결 대상

   ### 오래된 정보
   - [페이지명]: 내용

   ### 미생성 개념
   - [개념명]: 언급된 페이지 목록

   ### 누락된 교차 참조
   - [페이지A] <-> [페이지B]
   ```

4. **수정 적용**
   - 사용자 확인 후 수정을 진행합니다. (또는 `--auto` 플래그 시 자동 수정)
   - 명확한 오류(깨진 링크, frontmatter 누락)는 즉시 수정합니다.
   - 모순이나 내용 변경은 사용자에게 먼저 보고합니다.

5. **log.md 업데이트** (최상단에 prepend)
   ```
   ## [오늘날짜] lint
   - 점검 페이지 수: N
   - 발견된 문제: [요약]
   - 수정된 항목: [목록]
   ```
